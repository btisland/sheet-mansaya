📌 先了解 LINE 的兩個後台

LINE 設定會用到兩個不同的網站，別搞混了：

LINE Developers Console developers.line.biz

技術後台，給開發者用。負責 API 設定，在這裡取得 Channel Access Token 和 Your user ID。

LINE Official Account Manager manager.line.biz

管理後台，給小編或管理者用。負責 Bot 日常行為，在這裡建立官方帳號、啟用 Messaging API。

兩個後台管理的是同一個 LINE 官方帳號的不同面向。設定過程中你會在兩個後台之間切換，這是正常的。

A  ·  進入 LINE Developers Console

1

前往 developers.line.biz，點「Log in to Console」

在瀏覽器輸入 developers.line.biz，點右上角「Log in to Console」按鈕。

2

用 LINE 帳號登入

選擇「LINE帳號」登入方式，輸入你的 LINE 帳號和密碼。

3

建立 Provider，填入名稱後點「Create」

登入後如果還沒有 Provider，點「Create a new provider」，輸入一個名稱（例如你的名字或品牌），點「Create」。Provider 是你所有 LINE Bot 的「管理容器」。

B  ·  建立 LINE 官方帳號

4

在 Provider 頁面，選「Create a Messaging API channel」

Provider 建立後，你會看到幾個 channel 類型，點「Create a Messaging API channel」。

5

系統提示需要先建立官方帳號 → 點「Create a LINE Official Account」

LINE 目前需要透過 Official Account Manager 建立帳號，再回來連結。點綠色按鈕前往建立。

6

填入帳號資訊，點「確定」

填入 ① 帳號名稱（這會是 Bot 在 LINE 上的顯示名稱）、② 電子郵件、③ 國家、④ 業種，然後點⑤「確定」。

7

確認輸入內容，點「完成」

確認填入的資訊正確，點「完成」。

8

官方帳號建立完成 → 點「稍後進行認證（前往管理畫面）」

看到「您的 LINE 官方帳號已建立完成」，向下捲動點「稍後進行認證（前往管理畫面）」，進入 Official Account Manager。

C  ·  完成 Official Account Manager 初始設定

9

同意使用資訊 → 點「同意」

進入 Official Account Manager 後，第一次會出現資料使用同意書，點「同意」。

10

服務條款更新通知 → 點「了解並繼續使用」

接著會出現服務條款更新通知，點綠色按鈕繼續。

11

進入管理後台，關閉歡迎彈窗

進入 Official Account Manager 主頁，會跳出「開始經營帳號前」引導彈窗，點右上角 × 關閉即可。

12

設定 → 回應設定，關閉「加入好友的歡迎訊息」和「自動回應訊息」

點右上角「⚙️ 設定」→「回應設定」，找到 ③「加入好友的歡迎訊息」和 ④「自動回應訊息」，把兩個開關都關閉。這樣 Bot 才不會自動回覆用戶的訊息。

D  ·  啟用 Messaging API

13

設定 → Messaging API → 點「啟用 Messaging API」

在「設定」左側選單點 ① Messaging API，再點 ② 綠色按鈕「啟用 Messaging API」。

14

選擇服務提供者（Provider），點「同意」

選取你剛才在 Developer Console 建立的 Provider（如 SmartBot），點「同意」。

15

隱私權政策填寫（可略）→ 點「確定」；確認啟用 → 點「確定」

隱私權政策欄位可以留空，直接點「確定」。接著確認帳號名稱和 Provider 名稱無誤，再點「確定」完成啟用。

E  ·  回到 Developer Console，取得 Token 和 User ID

❓ Channel Access Token 和 User ID 是什麼？

Channel Access Token：Bot 的「API 密鑰」。讓你的 Google 試算表程式能代表這個 Bot 對外發送訊息。就像 Bot 的密碼，拿著它才能呼叫 LINE API。不可分享給他人。

Your user ID：你的 LINE 帳號在這個 Bot 中的「收件地址」。程式需要知道要把訊息發給誰，這個 ID 就是你的識別碼。每個用戶對每個 Bot 的 ID 都不同。
16

回到 developers.line.biz，進入剛建立的 Channel → 點「Messaging API」分頁

回到 LINE Developers Console，在左側 Providers 找到你的 Provider，點進去看到剛建立的 Channel，進入後點「Messaging API」分頁。

17

在「Channel access token」區塊，點「Issue」取得 Token，複製貼下

找到「Channel access token (long-lived)」，點「Issue」按鈕產生，再點旁邊的複製圖示。這串 Token 等下要填入 Sheet 的 B30。

18

點「Basic settings」分頁，找到「Your user ID」，複製

切換到「Basic settings」分頁，向下捲動找到「Your user ID」欄位，複製裡面的 ID（格式類似 Uxxxxxxxxxxxxxxxxx）。這個 ID 等下要填入 Sheet 的 B31。

F  ·  填入 Sheet 並測試

19

在設定頁填入 Token（B30）和 User ID（B31），點「立即抓取並推播」

回到 Google 試算表，在 ① B30 貼上 Channel Access Token，② B31 貼上 User ID，然後點 ③「▶ 立即抓取並推播」測試。

20

看到完成訊息，確認 LINE 收到推播

出現「✅ 完成！共推播 XX 則新文章。」後，打開 LINE 確認 Bot 是否傳來新聞摘要卡片。
