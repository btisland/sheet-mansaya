📌 先了解 LINE 的兩個後台

LINE 設定會用到兩個不同的網站，別搞混了：

LINE Developers Console developers.line.biz

技術後台，給開發者用。負責 API 設定，在這裡取得 Channel Access Token 和 Your user ID。

LINE Official Account Manager manager.line.biz

管理後台，給小編或管理者用。負責 Bot 日常行為，在這裡建立官方帳號、啟用 Messaging API。

兩個後台管理的是同一個 LINE 官方帳號的不同面向。設定過程中你會在兩個後台之間切換，這是正常的。

A  ·  進入 LINE Developers Console

1

前往 developers.line.biz，點「Log in to Console」

在瀏覽器輸入 developers.line.biz，點右上角「Log in to Console」按鈕。

2

用 LINE 帳號登入

選擇「LINE帳號」登入方式，輸入你的 LINE 帳號和密碼。

E  ·  回到 Developer Console，取得 Token 和 User ID

❓ Channel Access Token 和 User ID 是什麼？

Channel Access Token：Bot 的「API 密鑰」。讓你的 Google 試算表程式能代表這個 Bot 對外發送訊息。就像 Bot 的密碼，拿著它才能呼叫 LINE API。不可分享給他人。

Your user ID：你的 LINE 帳號在這個 Bot 中的「收件地址」。程式需要知道要把訊息發給誰，這個 ID 就是你的識別碼。每個用戶對每個 Bot 的 ID 都不同。
16

回到 developers.line.biz，進入剛建立的 Channel → 點「Messaging API」分頁

回到 LINE Developers Console，在左側 Providers 找到你的 Provider，點進去看到剛建立的 Channel，進入後點「Messaging API」分頁。

17

在「Channel access token」區塊，點「Issue」取得 Token，複製貼下

找到「Channel access token (long-lived)」，點「Issue」按鈕產生，再點旁邊的複製圖示。這串 Token 等下要填入 Sheet 的 B30。

18

點「Basic settings」分頁，找到「Your user ID」，複製

切換到「Basic settings」分頁，向下捲動找到「Your user ID」欄位，複製裡面的 ID（格式類似 Uxxxxxxxxxxxxxxxxx）。這個 ID 等下要填入 Sheet 的 B31。

常見問題

Q：為何在 Sheet 按右鍵，卻看不到「建立副本」？

這是因為還沒有登入 Google 帳號，請先登入再試一次。

Q：為什麼 Google 說「未驗證」，這樣安全嗎？

安全。這個警告代表工具尚未完成 Google 官方驗證流程，不代表有安全問題。授權畫面中你可以確認工具只存取 Sheet、Gmail 和外部服務，不會碰其他資料。

Q：為什麼我的 Email 沒有收到？

請確認：① B26 的 Email 地址填寫正確；② 查看垃圾郵件資料夾；③ 重新點「立即抓取並推播」，確認出現「✅ 完成！」訊息。如果訊息顯示「今天沒有新文章」，是因為關鍵字對應的新聞已推播過，可以先換個新的關鍵字測試。

Q：關鍵字填中文還是英文？

中英文都支援。追蹤台灣公司建議填中文（如「台積電」），追蹤國際品牌可填英文（如「Apple」、「NVIDIA」）。

Q：LINE 的 Token 跟 User ID 哪裡找？

Token 在 LINE Developers Console → Channel → Messaging API 分頁的「Channel access token」區塊；User ID 在「Basic settings」分頁的「Your user ID」欄位。詳見步驟 5-E。

Q：怎麼暫停或停止每日推播？

點「📡 新聞雷達」→「🔔 每日推播排程：開啟中（點此關閉）」即可關閉。之後想重新開啟再點一次即可。
